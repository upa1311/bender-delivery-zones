# Админский экран «Зоны и тарифы» — product contract

> Требования на будущее. Код в Direct **не интегрирован**, денежные значения
> **не назначены**, утверждённое количество зон **K=4 не меняется**.

## Основной принцип

Источник истины — **точный адрес**, а не название улицы.

Зона определяется по `settlement + district (при необходимости) + street + housenumber`.

Одна улица может иметь дома в разных зонах. Поэтому запрос **только по названию
улицы не возвращает один `zone_id`** — он возвращает `status: "ambiguous_street"`
и список домов с их зонами. Поведение зафиксировано в
[`schemas/address-zone-lookup.schema.json`](../../schemas/address-zone-lookup.schema.json).

Одинаковые названия улиц в разных населённых пунктах — **разные улицы**
(`улица Ленина` есть в Гиске, Парканах и Бендерах). Название улицы никогда не
содержит населённый пункт; уточнение добавляется только для отображения.

## 1. Поиск точного адреса

| режим | что вводит админ | что возвращает |
|---|---|---|
| `exact_address` | посёлок + улица + дом | одна карточка адреса с зоной |
| `street_only` | только улица | `ambiguous_street` + дома по зонам |
| `free_text` | «Ленина 15» | варианты по населённым пунктам |

При вводе «Ленина» показываются **отдельные варианты**, улица и место на разных
строках:

```
улица Ленина        улица Ленина        улица Ленина
Гиска               Парканы             Бендеры
```

## 2. Карточка адреса

- полный адрес — `Гиска, улица Ленина, дом 15`;
- населённый пункт и район отдельными полями;
- тарифная зона (`zone_id`, `zone_name`, границы диапазона в км);
- статус доставки — `standard` / `low_density` / `no_delivery` / `excluded` / `disputed`;
- OSRM-расстояние и время;
- центральный и BAM-маршрут отдельно (`assignment_basis.central_origin`, `.bam_origin`);
- **основание назначения зоны** — ожидаемые км с весами 0.85/0.15, худший источник,
  tier, правило;
- версия набора зон (`zone_dataset_version`).

## 3. Экран улицы

Открыв улицу, админ видит:

- какие **дома** относятся к каждой зоне (точные номера, а не только диапазон);
- **неадресные объекты** отдельным списком (жилые здания без `addr:housenumber`);
- **исключённые адреса** с причиной (`unreachable_by_osrm`,
  `no_serviceable_street_within_threshold`, `address_inside_nonresidential_building`,
  `outside_service_area`);
- **Tier C / no_delivery** отдельным блоком.

Сжатые диапазоны домов помечаются `ranges_are_exact`. Если один номер дома
встречается в разных зонах, диапазон **не** называется точным.

## 4. Карта

Карта четырёх зон (Zone 1…Zone 4): взаимоисключающие полигоны, точки адресов,
слои `no_delivery`, исключений и «нет адресных данных».

## 5. Фильтры

`Zone 1–4` · населённый пункт · район · улица · `no_delivery` · спорные адреса
(`disputed`: расхождение координат одного адреса, конфликт дублей, разрезанные
улицы).

## 6. Повторяющиеся названия

В любом списке, CSV, попапе и результате поиска используется
`display_address_ru`, поэтому перепутать нельзя:

- `улица Ленина (Гиска)`
- `улица Ленина (Парканы)`
- `улица Ленина (Бендеры)`

Внутри Бендер уточнение по району: `улица Энгельса (Липканы)` и
`улица Энгельса (Бендеры, другой район)`. Номер тарифной зоны **не** используется
для различения адресов.

## Версионирование

Каждый ответ несёт `zone_dataset_version`. Смена набора зон не меняет
`canonical_address_key` — идентичность адреса не зависит от тарификации.
