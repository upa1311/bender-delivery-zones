# Экран водителя — объяснение расчёта заказа

> Read-only contract. В Direct **не интегрировано**, цены **не назначены**,
> `K=4` не меняется.

## Задача

Водитель должен понимать, **почему** заказ стоит столько, сколько показано, без
обращения в поддержку. Экран только читает снапшот расчёта — он ничего не
пересчитывает.

## Что видит водитель

| поле | источник |
|---|---|
| адрес ресторана | `origin.full_address_ru` |
| `origin_zone` | `origin_zone_id` + название |
| адрес клиента | `destination.full_address_ru` |
| `destination_zone` | `destination_zone_id` + название |
| маршрут, км | `route_distance_km` (OSRM, реальная дорога) |
| применённая строка матрицы | `Zone X → Zone Y`, `tariff_version` |
| стоимость доставки для клиента | `customer_delivery_fee` |
| выплата водителю | `courier_payout` |
| доплаты | `calculation_components` с `applies_to: courier` |
| итоговая выплата | сумма компонентов курьера |
| версия тарифа | `tariff_version` + `zone_dataset_version` |

**Клиентская цена и выплата водителю хранятся и показываются раздельно.** Одно
никогда не выводится из другого на экране.

## Пример

```
Ресторан: Zone 1
Клиент:   Zone 3
Маршрут:  5.4 км
Тариф:    Zone 1 → Zone 3
Стоимость доставки для клиента: —
Выплата водителю:               —
```

Прочерки — это текущее состояние: денежные значения не назначены, пока владелец
не предоставит тарифы такси. Поля существуют в схеме и остаются `null`.

## Ресторан и клиент в разных зонах

Матрица направленная: `origin_zone_id → destination_zone_id`, 4 × 4 = 16 строк.
Заказ из Zone 1 в Zone 3 и заказ из Zone 3 в Zone 1 — разные строки.

## Аудит и неизменность

При расчёте заказа сохраняется снапшот
([`schemas/order-pricing-snapshot.schema.json`](../../schemas/order-pricing-snapshot.schema.json)):

`zone_dataset_version`, `tariff_version`, `origin_zone_id`,
`destination_zone_id`, `route_distance_km`, `route_duration_min`,
`customer_delivery_fee`, `courier_payout`, `calculation_components`.

Снапшот помечен `immutable: true`. **Позднейшее изменение зон или тарифов не
меняет историю завершённого заказа** — старые заказы продолжают показывать те
зоны, тариф и маршрут, которые действовали в момент расчёта.

Дополнительно фиксируется движок маршрутизации (версия OSRM, SHA-256 профиля
`car.lua` и исходного PBF), чтобы километраж заказа можно было воспроизвести.
